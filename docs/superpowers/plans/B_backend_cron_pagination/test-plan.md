# 阶段B测试计划（后端 Cron + Products 分页 + 前端联动）

## 1. 范围与目标
- 验证 `crawl_cron` 配置、APScheduler 热更新、products 服务端分页与前端真实对接行为。
- 确保阶段A已有功能不回归。

## 2. 功能用例
| 编号 | 场景 | 操作 | 预期结果 |
|---|---|---|---|
| B-01 | /config 读取 | 调用 GET `/config` | 返回 `crawl_cron`、`crawl_timezone` 字段 |
| B-02 | /config 全量更新 | 调用 POST `/config` | 兼容旧行为并可更新新字段 |
| B-03 | /config 部分更新 | 调用 PATCH `/config` 仅修改 `crawl_cron` | 返回 200 且配置生效 |
| B-04 | cron 非法校验 | PATCH 非法 cron | 返回 422，错误信息可读 |
| B-05 | 调度初始化 | 启动应用 | scheduler 启动并注册唯一 job |
| B-06 | 调度热更新 | 更新 `crawl_cron` | 无需重启，job 被替换并按新表达式触发 |
| B-07 | 并发保护 | 人工制造上次任务未结束场景 | 下一次触发被跳过并记录日志 |
| B-08 | products 分页 | GET `/products?page=1&page_size=15` | 返回分页结构与 `items` |
| B-09 | products 筛选分页 | 组合筛选 + 分页参数 | 数据与元信息一致 |
| B-10 | 前端定时配置真实读写 | 定时页读取并保存 | GET 展示后端值，PATCH 保存成功 |
| B-11 | 草稿兜底 | 修改但不保存后刷新 | 可恢复 localStorage 草稿或丢弃 |
| B-12 | 商品页服务端分页 | 前端翻页/筛选 | 请求带分页与筛选参数且 UI 正确 |

## 3. 边界与异常用例
| 编号 | 场景 | 预期结果 |
|---|---|---|
| B-E01 | 配置不存在 | 返回默认配置或自动创建默认配置 |
| B-E02 | page 越界 | 返回空 `items` + 正常分页元信息 |
| B-E03 | 筛选无结果 | `total=0`，前端正常空态 |
| B-E04 | 调度更新与执行冲突 | 当前执行可完成，新配置下次触发生效 |
| B-E05 | 批量删除后页空 | 前端自动回退上一页并刷新 |
| B-E06 | 5xx/超时 | 前端显示可重试，后端错误日志可定位 |
| B-E07 | Redis/DB 短暂不可用 | 接口返回明确失败，不出现静默异常 |

## 4. 回归用例
| 编号 | 场景 | 预期结果 |
|---|---|---|
| B-R01 | 手动抓取 | POST `/crawl/crawl-now` 仍可用 |
| B-R02 | 健康检查 | GET `/health` 正常 |

- 核心用例 `B-01` 到 `B-12` 全部通过。
- 边界用例无 P0/P1 缺陷。
- 回归用例全通过，且文档已更新（README / ARCHITECTURE）。
